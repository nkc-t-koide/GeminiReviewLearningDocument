Workflowファイルを作る

AIレビューを自動で起動するために、GitHub Actions のWorkflowファイルを作ります。

12345678

YAMLファイルを作る

どこに作る？

GitHub Actions の設定ファイルは、リポジトリ内の .github/workflows/ フォルダに作成します。
今回は、以下の場所に ai-review.yml を作ります。

repo/
└─ .github/
   └─ workflows/
      └─ ai-review.yml
補足： GitHubは .github/workflows/ に置かれたYAMLファイルを、自動処理の設定として認識します。

実際のコードを書く

以下の内容を .github/workflows/ai-review.yml に書きます。

name: AI PR Review

on:
  pull_request:
    types: [opened, synchronize, reopened]

permissions:
  contents: read
  pull-requests: write
  issues: write

jobs:
  ai-review:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: "3.11"

      - name: Install dependencies
        run: |
          pip install google-genai requests

      - name: Run AI review
        env:
          GEMINI_API_KEY: ${{ secrets.GEMINI_API_KEY }}
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          GITHUB_REPOSITORY: ${{ github.repository }}
          PR_NUMBER: ${{ github.event.pull_request.number }}
          BASE_SHA: ${{ github.event.pull_request.base.sha }}
          HEAD_SHA: ${{ github.event.pull_request.head.sha }}
        run: |
          python scripts/ai_review.py

用語Tips

はじめに確認

このページで出てきた言葉を、先にざっくり確認しておきましょう。

TIP 01

YAMLファイルとは

設定を書くためによく使われるテキストファイルです。GitHub Actionsでは、いつ・どの環境で・どの処理を動かすかをYAMLで書きます。

TIP 02

Workflowとは

GitHub Actionsで実行する自動処理のまとまりです。PR作成やpushなどをきっかけに実行できます。

TIP 03

runnerとは

Workflowを実行するための作業環境です。今回はGitHubが用意するUbuntu環境でPythonを実行します。
